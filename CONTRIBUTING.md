# Contributing

This repository welcomes contributions that improve the quality, clarity, and maintainability of the archive.

## Good Contributions

- README, navigation, and repository metadata improvements
- Corrections to analysis documents when backed by clear evidence
- Translation fixes and terminology cleanup
- Build notes, reproducibility notes, and local workflow improvements
- Broken link fixes, typo fixes, and issue template improvements

## Out of Scope

- Uploading unpublished or proprietary upstream materials
- Removing attribution, notice, or rights-related context
- Re-licensing the extracted upstream source
- Using this repository as an official Anthropic support channel
- Submitting changes that imply Arain owns the upstream extracted source

## Pull Request Expectations

- Keep changes scoped to repository maintenance, docs, or research quality
- Explain factual corrections with references, screenshots, or code pointers when possible
- Do not add secrets, tokens, personal data, or non-public materials
- Preserve upstream attribution and repository notice language
- Update [README.md](README.md), [NOTICE.md](NOTICE.md), or related docs when public-facing behavior changes

## Before You Open a PR

- Use the issue templates when the change starts from a bug report or correction
- Read [SECURITY.md](SECURITY.md) for sensitive issues
- Read [NOTICE.md](NOTICE.md) for rights, attribution, and takedown context

## Language

English or Chinese contributions are both fine. Clear, evidence-based changes are preferred over large speculative rewrites.

# 贡献说明

欢迎提交能够提升本仓库质量、清晰度和可维护性的改动。

## 欢迎的改动

- README、导航结构和仓库元信息优化
- 有明确证据支撑的分析文档修正
- 翻译修正与术语统一
- 构建说明、复现说明和本地工作流改进
- 失效链接、错字以及 issue 模板优化

## 不接受的内容

- 上传未公开或专有的上游材料
- 删除归属说明、声明文件或权利相关信息
- 试图为提取出的上游源码重新授予许可证
- 把本仓库当作 Anthropic 官方支持渠道
- 提交会让人误以为 Arain 拥有上游提取源码权利的改动

## PR 约定

- 请把改动范围控制在仓库维护、文档质量或研究资料质量之内
- 涉及事实修正时，尽量提供引用、截图或代码定位
- 不要提交密钥、令牌、个人隐私数据或非公开材料
- 保留上游归属与仓库声明相关内容
- 如果改动影响公开展示，请同步更新 [README.md](README.md)、[NOTICE.md](NOTICE.md) 或相关文档

## 提交前建议

- 如果改动来源于 bug、文档修正或仓库改进建议，优先使用 issue 模板
- 敏感问题请先阅读 [SECURITY.md](SECURITY.md)
- 权利、归属和下架相关事项请先阅读 [NOTICE.md](NOTICE.md)
